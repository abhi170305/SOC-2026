## Goal

Solve the first inverse problem with a PINN — recovering an unknown PDE parameter from sparse,
noisy observations by treating the parameter as a trainable variable alongside the network weights.

## Problem — Diffusion-Reaction

```
uₜ = D·u_xx − k·u,   x ∈ [0,1],  t ∈ [0,1]
D = 0.001  (known)
k = 0.01   (TRUE value — unknown to the model, to be recovered)
```

Ground-truth data is generated by solving the PDE with `k = 0.01` (finite differences +
`solve_ivp`), then 100 random `(x,t)` points are sampled as sparse observations. The recovered
parameter `k` is a `torch.nn.Parameter` trained jointly with the network via the hybrid loss:

```
L_total = L_pde + L_data
```

## Tasks & Results

### Part A — Baseline recovery (noise-free)

Recovered **k = 0.01000** vs true **0.01** — **relative error 0.03%**. Near-perfect recovery.
The parameter trajectory converges cleanly to the true value over 10,000 epochs.

### Noise sweep

Recovery repeated at observation noise levels σ ∈ {0%, 1%, 5%}:

| Noise Level | Recovered k | Relative Error |
|---|---|---|
| 0% | 0.00953 | **4.74%** |
| 1% | 0.00693 | **30.73%** |
| 5% | 0.00244 | **75.60%** |

**Key finding:** Parameter recovery degrades sharply with noise. At 0% noise the estimate is
close (4.74% error), but even 1% observation noise pushes the error to ~31%, and 5% noise makes
recovery unreliable (76% error, k off by 4×). The PDE residual acts as a regulariser but cannot
fully compensate once noise dominates the data term.

### Bonus — fewer observations

| Observations | Recovered k | Relative Error |
|---|---|---|
| 100 | 0.01000 | 0.03% |
| 10 | 0.00860 | 13.98% |

With only 10 observation points instead of 100, error rises to ~14% — fewer data points weaken
the anchoring of the fit, so the optimiser can settle on a `k` that satisfies the PDE almost as
well but deviates from the truth.

## Key Concept

In inverse problems the unknown parameters become **part of the optimisation** — the PINN
simultaneously solves the PDE *and* discovers what the PDE is. This is where PINNs genuinely
outperform classical methods, which typically need complete observational data. Structurally,
recovering ν/ρ in Navier-Stokes (Raissi Part II) is harder: coupled nonlinear equations, indirect
observation (concentration, not velocity), and multiple parameters with identifiability issues.
